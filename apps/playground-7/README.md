# playground-7 — способ «фабрика плагина» (`createGranularity`)

Это приложение демонстрирует **третий способ** интеграции `@feugene/granularity`
в Vue 3 — через **фабрику плагина** `createGranularity(...)`, экспортируемую из
отдельного sub-path `@feugene/granularity/vue`.

Фабрика — это лёгкий адаптер, который **не импортирует ни одного компонента
или директивы сам**: всё приходит параметрами от пользователя. За счёт этого
сохраняются две ключевые фишки гранулярности:

1. **Tree-shaking** — в бандл попадает ровно то, что вы передали в опции фабрики.
2. **Независимые sub-path impорт-ы** — CSS и JS компонента подтягиваются только
   по явному импорту `@feugene/granularity/components/<Name>`.

## Что включает приложение

Приложение использует **ровно два компонента** и **одну директиву**:

- `DsButton` — кнопка «Добавить».
- `DsInput` — поле ввода.
- `v-hotkey` — слушает нажатие `Enter` внутри `DsInput` и добавляет значение в
  список.

Всё регистрируется глобально через `createGranularity(...)` в `src/main.ts`.

## Как это устроено

`src/main.ts`:

```ts
import { createGranularity } from '@feugene/granularity/vue'
import { DsButton } from '@feugene/granularity/components/DsButton'
import { DsInput }  from '@feugene/granularity/components/DsInput'
import { vHotkey }  from '@feugene/granularity/directives/hotkey'

const Granularity = createGranularity({
  // Достаточно передать сами компоненты — имя регистрации Vue возьмёт
  // из `__name` (у SFC оно подставляется из имени файла) или из `name`.
  // Если нужно зарегистрировать под другим именем — используйте форму
  // `{ name: 'MyButton', component: DsButton }`.
  components: [DsButton, DsInput],
  directives: [{ name: 'hotkey', directive: vHotkey }],
})

createApp(App).use(Granularity).mount('#app')
```

Фабрика принимает компоненты в любой из трёх форм:

- **Голый компонент** (`DsButton`) — имя берётся из `__name`/`name`.
- **`{ install(app) { … } }`** — компонент сам управляет регистрацией (удобно
  для собственных per-component плагинов).
- **`{ name, component }`** — явное переопределение имени регистрации.

Обратите внимание:

- **Сам модуль `@feugene/granularity/vue` пустой** — это просто фабрика + хелперы.
  Он **не импортирует `index.ts` пакета** и не тянет за собой ничего
  лишнего.
- **Компоненты и директивы импортируются отдельно** через sub-path — именно эти
  импорты определяют, что Rollup/Rolldown оставит в бандле.
- **Стили подключаются так же гранулярно**: только `foundation.css` +
  `components/DsButton/styles.css` + `components/DsInput/styles.css`.

Проверить это можно в продакшн-сборке: в чанке `granularity` должны встретиться
только `DsButton`, `DsInput` и `hotkey` из пакета, всё остальное отсутствует.

## Когда выбирать этот способ

- Когда хочется **одной строкой** `app.use(Granularity)` в `main.ts` (не писать
  десятки `app.component(...)` или `app.use(...)` вручную).
- Когда нужно в одном месте **прокинуть общие настройки** через `provides`
  (например, i18n-адаптер, тему) и навесить `globalProperties`.
- Когда в проекте **много компонентов/директив из granularity**, и их список
  стабильный — удобно держать конфиг в одном файле.

## Нюансы и подводные камни

- 🟢 **Tree-shaking сохраняется** ровно до тех пор, пока вы сами не импортируете
  лишнего. Если где-то в коде появится `import { X } from '@feugene/granularity'`
  (корневой импорт), это может затащить в бандл весь `index.ts`. Поэтому для
  «фабричного» способа рекомендуется импортировать **только из sub-path**
  (`@feugene/granularity/components/…`, `/directives/…`, `/composables/…`,
  `/vue`).
- 🟡 **Глобальная регистрация** через `app.component('DsX', DsX)` «прячет»
  компонент от статического анализа шаблонов. Volar перестаёт подсвечивать тип.
  Если это важно, используйте declaration merging `GlobalComponents` в
  отдельном `.d.ts` приложения, либо подключайте компоненты локально
  (через `<script setup>` + `import`) без глобальной регистрации.
- 🟡 **Стили придётся подключать самостоятельно** — фабрика их не знает. Один
  раз `foundation.css` и по одному `components/<Name>/styles.css` на каждый
  нужный компонент. Либо используйте общий `@feugene/granularity/styles.css`,
  но тогда потеряете часть экономии CSS.
- 🟢 **Совместимо с локальным импортом.** Если где-то понадобится компонент,
  который не зарегистрирован через фабрику — просто импортируйте его локально
  в `<script setup>`. Ничего настраивать не нужно.
- 🟢 **Совместимо с авто-импортом (`unplugin-vue-components`).** Вы можете
  настроить резолвер и не регистрировать компоненты в фабрике вовсе — тогда
  фабрика пригодится только для директив/`provides`. См. пример в `playground-8`.
- 🔴 **Не импортируйте из фабрики «всё подряд».** Сам смысл sub-path `./vue`
  именно в том, что он пустой. Любая попытка добавить внутрь него
  «удобный список всех компонентов» превращает фабрику в монолитный плагин и
  ломает гранулярность.
